# Инструкции для агента: Лучшие практики разработки интерактивных презентаций и дашбордов

## Обзор
Этот файл содержит инструкции для AI-агента по разработке и поддержке проектов типа интерактивных презентаций, дашбордов и веб-приложений, подобных PIN-UP 10Y Merch Intelligence Report.

## Структура проекта
Проект организован по модульной структуре:
- `index.html` - основной HTML-файл
- `sections/` - отдельные HTML-файлы для секций (dashboard.html, explorer.html, strategy.html)
- `assets/css/styles.css` - кастомные стили
- `assets/js/script.js` - логика и данные
- `.instructions.md` - этот файл с инструкциями

## Лучшие практики

### 1. Архитектура и Структура
- **Модульность**: Разделяйте большой HTML-файл на секции (sections/), компоненты и отдельные файлы для лучшей поддерживаемости.
- **Компонентный подход**: Используйте переиспользуемые компоненты (например, карточки, модалы) для consistency.
- **Разделение concerns**: Отделяйте HTML, CSS и JS; используйте внешние файлы для стилей и скриптов при росте проекта.

### 2. Код и Комментарии
- **Добавляйте комментарии**: В HTML, CSS и JS объясняйте назначение секций, функций и сложных частей кода.
- **Семантический HTML**: Используйте правильные теги (nav, main, section, article) для accessibility и SEO.
- **Читаемый код**: Форматируйте код, используйте осмысленные имена переменных и функций.

### 3. Производительность
- **Оптимизация загрузки**: Минимизируйте CSS/JS, используйте CDN для библиотек (Tailwind, Chart.js).
- **Ленивая загрузка**: Загружайте контент по требованию (например, секции или данные).
- **Responsive design**: Обеспечьте адаптивность на всех устройствах с помощью media queries.

### 4. Пользовательский опыт (UX)
- **Интерактивность**: Добавляйте плавные переходы, анимации и feedback (hover, active states).
- **Accessibility**: Используйте ARIA-атрибуты, контрастные цвета, клавиатурную навигацию.
- **Мобильная адаптация**: Тестируйте на мобильных устройствах, используйте touch-friendly элементы.

### 5. Данные и Логика
- **Структурированные данные**: Храните данные в JSON или массивах для легкого обновления.
- **Валидация**: Проверяйте ввод пользователя и данные перед обработкой.
- **Безопасность**: Избегайте inline скриптов с пользовательским вводом; используйте CSP.

### 6. Инструменты и Фреймворки
- **Tailwind CSS**: Для быстрого прототипирования и consistent дизайна.
- **Chart.js**: Для визуализации данных; настраивайте опции для лучшей читаемости.
- **Vanilla JS**: Для простых взаимодействий; переходите на фреймворки (React, Vue) при сложности.

### 7. Тестирование и Поддержка
- **Кросс-браузерность**: Тестируйте в Chrome, Firefox, Safari, Edge.
- **Отладка**: Используйте console.log, breakpoints; проверяйте на ошибки в DevTools.
- **Документация**: Ведите README с инструкциями по запуску и разработке.

### 8. Специфично для презентаций
- **Навигация**: Простая и интуитивная; используйте tabs или sidebar для больших объемов.
- **Визуализация**: Выбирайте подходящие графики (radar для баланса, doughnut для распределения).
- **Фильтрация**: Добавляйте поиск и фильтры для больших датасетов.
- **Экспорт**: Рассмотрите возможность экспорта в PDF или изображения.

### 9. Этика и Лучшие практики
- **Конфиденциальность**: Не храните личные данные; используйте HTTPS.
- **Производительность**: Оптимизируйте для быстрой загрузки, особенно на мобильных.
- **Обновляемость**: Делайте код гибким для добавления новых секций или данных.
